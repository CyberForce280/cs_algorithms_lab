🔄 الاستدعاء الذاتي — Recursion

الاستدعاء الذاتي هو قدرة الدالة على استدعاء نفسها. يحتاج لحالة أساسية (Base Case) لإيقاف التكرار.

📖 المفهوم الأساسي

  • الحالة الأساسية (Base Case): الشرط الذي يوقف الاستدعاء — مثل n=0 في المضروب
  • الجزء الاستقرائي: الاستدعاء الذاتي بمعامل أصغر — factorial(n-1)
  • كل استدعاء يُحفظ في Stack حتى تُحل الحالة الأساسية ثم تُعاد القيم
// حساب مضروب n بالاستدعاء الذاتي
int factorial(int n)
{
    if (n > 1)
        return n * factorial(n - 1);  // الجزء الاستقرائي
    else
        return 1;                          // الحالة الأساسية
}

🕸️ الجراف — Graph

الجراف هو هيكل بيانات غير خطي يتكون من مجموعة رؤوس (Vertices) وحواف (Edges) تربطها.

📐 التعريف الرياضي

الجراف G = (V, E) حيث:
• V = مجموعة الرؤوس (Vertices/Nodes)
• E = مجموعة الحواف (Edges/Arcs)
• كل حافة تربط رأسين: (startVertex, endVertex)

🏷️ أنواع الجرافات

• Undirected: حواف غير موجهة (ثنائية الاتجاه)
• Directed: حواف موجهة (أحادية الاتجاه)
• Connected: كل رأسين مرتبطان بمسار
• Disconnected: بعض الرؤوس منفصلة

🎨 مرئي — جراف تفاعلي

اضغط أحد الأزرار لرسم مثال من الملزمة: V = {A,B,C,D,E}

📌 المصطلحات المهمة

  • Vertex (Node): عنصر بيانات منفرد في الجراف
  • Edge (Arc): الرابط بين رأسين، يُمثَّل كـ (A, B)
  • Degree: عدد الحواف المتصلة برأس معين
  • Path: تسلسل من الرؤوس والحواف من نقطة لأخرى

🌳 الأشجار — Trees

الشجرة هي جراف متصل لا يحتوي على دورات (Acyclic Connected Graph). إذا كان عدد العقد N فعدد الحواف N-1.

🌱 Root

العقدة الأولى، أصل الشجرة. كل شجرة لها root واحد فقط. عمقه = 0.

👨‍👦 Parent / Child

Parent = العقدة السابقة (الأم). Child = العقدة التابعة (الابن). كل عقد ما عدا الجذر هو Child.

🍃 Leaf

عقدة ليس لها أبناء. تُسمى أيضاً Terminal Node. ارتفاعها = 0.

🤝 Siblings

العقد التي لها نفس الأب تُسمى Siblings (أشقاء).

📏 Degree

عدد أبناء العقدة. Degree of Tree = أعلى درجة لأي عقدة في الشجرة.

🏔️ Height / Depth

Height: حواف من الورقة إلى العقدة (لأعلى). Depth: حواف من الجذر إلى العقدة (لأسفل).

🎨 مثال من الملزمة — شجرة بـ 11 عقدة

🔵 أزرق = عقدة عادية 🟢 أخضر = عقدة أب 🟠 برتقالي = ورقة (Leaf)

🌲 الشجرة الثنائية — Binary Tree

شجرة خاصة حيث كل عقدة لها على الأكثر ابنان: ابن أيسر وابن أيمن.

📐 خصائص رياضية

• Max nodes في المستوى L = 2^L
• Max nodes في شجرة ارتفاعها h = 2^(h+1) - 1
• عدد الأوراق = (عدد عقد بدرجة 2) + 1

🗂️ طرق التمثيل

• Array: الجذر T[1]، ابن أيسر T[2i]، ابن أيمن T[2i+1]
• Linked List: كل عقدة = {Left | Data | Right}
• المصفوفة تحتاج حجم 2^(n+1)-1

🚶 اجتياز الشجرة — Tree Traversal

الأنواع الثلاثة

  • In-Order: يسار → جذر → يمين → ينتج ترتيباً تصاعدياً في BST
  • Pre-Order: جذر → يسار → يمين → مفيد لنسخ الشجرة
  • Post-Order: يسار → يمين → جذر → مفيد لحذف الشجرة
اختر نوع الاجتياز...

🔢 تمثيل المصفوفة — Array Representation

مثال الملزمة: الجذر A في T[1]، ابن أيسر B في T[2]، ابن أيمن C في T[3]...

🔍 شجرة البحث الثنائية — BST

شجرة ثنائية خاصة: كل قيم الشجرة الفرعية اليسرى أصغر من الجذر، وكل قيم الشجرة الفرعية اليمنى أكبر منه.

🏗️ ابنِ شجرة BST الخاصة بك

أدخل أرقاماً لبناء الشجرة، أو اضغط "مثال الملزمة"...

🗑️ حالات الحذف في BST

  • Case 1 — ورقة: احذف مباشرة وضع الأب → NULL
  • Case 2 — ابن واحد: ربط الأب مباشرة بالابن
  • Case 3 — ابنان: استبدل بأكبر قيمة في الشجرة اليسرى، أو أصغر قيمة في اليمنى
// إدراج في BST — C++
struct Node {
    int data;
    Node* left;
    Node* right;
};

Node* insert(Node* root, int val) {
    if (root == nullptr) return new Node(val);
    if (val < root->data)
        root->left = insert(root->left, val);
    else
        root->right = insert(root->right, val);
    return root;
}

🫧 Bubble Sort — الترتيب الفقاعي

يقارن كل عنصرين متجاورين ويبدّلهما إذا كانا بالترتيب الخاطئ. الأكبر "يطفو" لأعلى في كل جولة.

📊 التعقيد الزمني

  • أسوأ حالة: O(n²)
  • أفضل حالة: O(n) مع التحسين
  • حالة متوسطة: O(n²)

🔄 كيف يعمل؟

  • حلقتان متداخلتان: i و j
  • كل جولة خارجية تضع أكبر عنصر في مكانه
  • n-1 جولة كافية لترتيب n عنصر
void bubbleSort(int data[], int n)
{
    int temp;
    for (int i = 0; i < (n-1); i++)           // n-1 جولة
    {
        for (int j = n-1; j > i; j--)         // من الآخر للأمام
        {
            if (data[j] < data[j-1])          // مقارنة
            {
                temp = data[j];
                data[j] = data[j-1];
                data[j-1] = temp;              // تبادل
            }
        }
    }
}

📥 Insertion Sort — الترتيب بالإدراج

يبني الجزء المرتّب تدريجياً بإدراج كل عنصر في مكانه الصحيح، مثل ترتيب ورق اللعب.

📊 التعقيد الزمني

  • أسوأ حالة: O(n²)
  • أفضل حالة (مرتّب): O(n)
  • فعّال لمصفوفات صغيرة أو شبه مرتّبة

🃏 الفكرة

  • اختر عنصراً (key)
  • قارنه بالعناصر المرتّبة قبله
  • أزح العناصر الأكبر يميناً
  • ضع key في المكان الصحيح
void insertionSort(int data[], int n)
{
    int i, key, j;
    for (i = 1; i < n; i++)
    {
        key = data[i];                              // العنصر المراد إدراجه
        for (j = i; j > 0 && key < data[j-1]; j--)
            data[j] = data[j-1];                  // إزاحة للأمام
        data[j] = key;                              // الموضع الصحيح
    }
}

⚡ Quick Sort — الترتيب السريع

يختار عنصراً محورياً (Pivot) ويقسم المصفوفة حوله، ثم يُطبَّق بشكل تكراري على الأجزاء. أسرع خوارزمية عملياً في المتوسط.

📊 التعقيد الزمني

  • متوسط: O(n log n)
  • أسوأ حالة: O(n²) إذا كان المحور الأسوأ دائماً
  • أفضل عملياً لأنه In-Place و Cache-Friendly

🎯 خطوات القسمة

  • اختر المحور Pivot (عادة العنصر الأول)
  • حرّك lower و upper نحو بعضهما
  • ضع ما < pivot يساراً وما > يميناً
  • ضع pivot في مكانه الصحيح
void quickSort(int data[], int first, int last)
{
    while (last > first) {
        int lower = first, upper = last;
        int bound = data[first];              // المحور = أول عنصر
        while (lower < upper) {
            while (data[upper] > bound) upper--;
            data[lower] = data[upper];
            while (lower < upper && data[lower] <= bound) lower++;
            data[upper] = data[lower];
        }
        data[lower] = bound;                  // ضع المحور في مكانه
        quickSort(data, first, lower-1);      // الجزء الأيسر
        first = lower + 1;                   // الجزء الأيمن
    }
}

🔎 البحث التسلسلي — Sequential Search

أبسط خوارزمية بحث: تبحث في كل عنصر من البداية حتى النهاية حتى تجد المطلوب.

📊 التعقيد

  • أسوأ حالة (لم يُوجد): O(n)
  • أفضل حالة (أول عنصر): O(1)
  • لا يشترط أن تكون المصفوفة مرتبة

✅ متى نستخدمه؟

  • المصفوفة صغيرة أو غير مرتّبة
  • البحث مرة واحدة فقط
  • بسيط وسهل التطبيق
void sequentialsearch(int arr[], int size, int k)
{
    int i, pos = -1;
    for (i = 0; i < size; i++)
    {
        if (arr[i] == k)
            pos = i;
    }
    if (pos == -1)
        cout << "العنصر غير موجود";
    else
        cout << "وُجد في الموضع " << pos;
}

⚡ البحث الثنائي — Binary Search

يقسّم المصفوفة المرتّبة نصفين في كل خطوة ويحدد في أي نصف يوجد العنصر. أسرع بكثير من التسلسلي.

📊 التعقيد

  • أسوأ حالة: O(log n)
  • أفضل حالة (المنتصف): O(1)
  • يشترط أن تكون المصفوفة مرتّبة!

🎯 الخوارزمية

  • mid = (lower + upper) / 2
  • إذا key == data[mid] → وُجد!
  • إذا key < data[mid] → ابحث يساراً (upper = mid-1)
  • إذا key > data[mid] → ابحث يميناً (lower = mid+1)
int binarySearch(int data[], int k, int lower, int upper)
{
    int pos = -1, mid;
    if (lower <= upper) {
        mid = (lower + upper) / 2;
        if (k == data[mid]) {
            return mid;                          // وُجد!
        } else if (k < data[mid])
            return binarySearch(data, k, lower, mid-1);  // يسار
        else
            return binarySearch(data, k, mid+1, upper); // يمين
    }
    return -1;                                 // لم يُوجد
}
